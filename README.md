O arquivo eouve.pdf foi fornecido para análise, estudo e coleta de dados. Após essa etapa, foi realizada a criação do banco de dados com base no Diagrama Entidade-Relacionamento (DER) previamente modelado, garantindo a correta organização, integridade e padronização das informações extraídas. Essa estruturação possibilitou a centralização dos dados da Ouvidoria em um ambiente relacional, facilitando consultas, análises e futuras manutenções.
Em seguida, os dados foram tratados e transformados utilizando bibliotecas da linguagem Python, como Pandas e NumPy, assegurando a consistência, limpeza e normalização das informações antes de sua inserção definitiva no banco de dados MySQL. Esse processo foi fundamental para eliminar inconsistências e preparar os dados para análises mais precisas.
Com o banco de dados estruturado e alimentado, a etapa final consistiu na integração com o Power BI, ferramenta responsável pela visualização das informações. Foram desenvolvidos dashboards interativos, gráficos e medidas personalizadas, permitindo uma análise clara e dinâmica das demandas registradas na Ouvidoria. Dessa forma, os dados brutos foram transformados em informações estratégicas, auxiliando gestores públicos na tomada de decisões baseadas em evidências.
A solução proposta adota uma abordagem Data-Driven, substituindo processos manuais e descentralizados por um fluxo automatizado, confiável e orientado à análise de dados. Como resultado, o projeto contribui para maior eficiência operacional, melhor compreensão dos problemas recorrentes da cidade e apoio direto à gestão pública na definição de ações prioritárias para a melhoria dos serviços prestados à população de Limeira.
